# 附录：术语与维护规则

## 1. 关键术语

- BFF：面向前端的聚合 API 层。
- SPU/SKU：商品抽象与库存销售单元。
- 逻辑删除：通过状态字段隐藏数据，而非物理删除。
- 证据链：支撑业务结论的代码/数据/日志/测试组合。

## 2. 维护规则

1. 每次业务链路变更，优先更新 `04_core_flows` 与 `06_data_flow`。
2. 每次后台变更，优先更新 `07_admin_ops`。
3. 每次形成结论，优先回填 `08_evidence`。
4. 每次阶段完成，根 README Changelog 追加“变更点/验证点/新增规律/遗留问题”。

## 3. 质量门槛

- 图文一致：图里的节点必须在正文有解释。
- 事实可追溯：每章至少 3 个事实引用。
- 文档可执行：每章至少 3 条可验证检查点。
